# 添加账号中文输出设计

## 目标

修复添加账号窗口把 ANSI 终端颜色控制码显示成 `90m`、`94m`、`0m` 等普通文字的问题，并将该窗口的设备登录提示中文化。

## 范围

- 自动删除添加账号流式输出中的 ANSI 控制序列。
- 将添加账号窗口中的“添加账号”“等待设备登录”“完成”“失败”“关闭”改为中文。
- 将 Codex 设备登录步骤及安全提醒翻译成中文。
- 登录网址和一次性验证码必须逐字保留，继续支持选择和复制。
- 仅修改添加账号窗口及其登录输出展示，不修改主窗口、切换账号窗口、移除账号窗口或 `codex-auth` 的实际登录行为。

## 设计

新增一个无状态的添加账号输出格式化器。它先删除 ANSI CSI/SGR 控制序列，再对已知的 Codex 设备登录提示逐行进行中文映射；无法识别的行只删除控制码，不翻译、不丢弃。包含有效期分钟数的已知提示保留原数值后翻译，网址、验证码和其他动态内容不参与替换。

格式化器只接入 `AccountDialogService.RunLoginAsync` 的显示回调，因此不会改变 `CommandResult`、认证事务、日志、账号快照或其他操作窗口。`OperationWindow` 接受每个实例自己的完成、失败和关闭文案；只有添加账号实例传入中文文案，移除账号实例继续使用现有英文文案。

## 错误与兼容性

- 空行保持为空行，维持提示的可读分段。
- 未知 ANSI SGR/CSI 控制序列应被删除；普通中英文文本保持原样。
- Codex CLI 文案未来变化时，未知文本仍会安全显示，只是不翻译。
- 格式化不得修改 URL、设备验证码或敏感信息脱敏顺序。

## 验证

- 单元测试复现截图中的 ANSI 文本，并证明输出中不再含控制序列或裸露的 `90m/94m/0m`。
- 单元测试覆盖所有已知英文登录提示的中文映射。
- 单元测试证明 URL、验证码、未知文本及空行保持不变。
- WPF 运行时测试验证添加账号窗口的标题、阶段、完成/失败和关闭按钮为中文。
- 完整 Release 测试、9 文件发布合同、稳定安装文件哈希及真实设备登录窗口进行回归验证。
