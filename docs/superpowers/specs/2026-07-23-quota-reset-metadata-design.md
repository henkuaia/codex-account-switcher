# 额度、重置次数与添加账号体验设计

## 背景

现有软件可手动刷新账号的周额度或月额度剩余百分比，但尚未显示 OpenAI 发放的重置次数，也没有记录每个账号单次周/月额度美元总值。添加账号还会先显示一次确认弹窗，再显示登录进度窗口，流程显得冗长。

GitHub 与 OpenAI 官方实现核对结果如下：

- `/backend-api/wham/usage` 可返回 `rate_limit_reset_credits.available_count`，表示当前可用重置次数。
- 官方只将当前可用次数定义为权威快照，不提供历史累计使用次数。
- 可选的 `individualLimit` 表示月度信用/支出上限，不等于周额度套餐的单周期美元总值。
- `credits.balance` 是额外 credits 余额，不等于套餐单周期额度。
- 公开项目中的 credits-to-USD 换算只能估算消费价值，不能反推出套餐基础额度。

## 目标

- 手动刷新额度时，同时显示当前可用重置次数。
- 显示每个账号的累计已使用重置次数，并明确标注为本机记录。
- 记录并显示每个账号的单次周/月额度美元总值，不把赠送重置次数乘入该金额。
- 当后端提供可信的月度 `individualLimit` 时，可将其作为官方月度上限显示；否则使用用户保存的单周期金额。
- 点击“+”后直接进入一个可关闭的登录进度窗口，移除前置确认弹窗。
- 不修改认证快照、对话历史、个性化配置或 `codex-auth` 账号注册表格式。

## 非目标

- 不根据可用次数的增减自动推断历史已用次数。
- 不增加“消耗重置券”功能。
- 不自动切换账号。
- 不把 `credits.balance`、累计 credits 消耗或估算美元值显示为套餐单周期总额度。
- 不扩大到其他页面的全面中文化或 UI 重构。

## 数据设计

### 服务器快照

扩展现有额度解析结果，读取：

- 长周期额度窗口：周期类型、剩余百分比、重置时间。
- `rate_limit_reset_credits.available_count`：当前可用重置次数。
- 可选 `individualLimit`：仅在后端明确提供时作为官方月度上限。

缺失字段显示“—”，不得按零处理。

### 本机账号元数据

新增软件自有文件：

`%LOCALAPPDATA%\CodexAccountSwitcher\account-metadata.json`

按稳定的 `AccountKey` 保存：

- `periodQuotaUsd`：用户记录的单次周/月额度美元值，可为空。
- `usedResetCount`：用户记录的累计已使用重置次数，默认 0，可修改。

该文件不保存令牌、密码、验证码或完整认证 JSON。账号被移除时暂不删除对应元数据，避免误操作后重新添加账号造成记录丢失。

保存采用临时文件替换，损坏或无法解析时不覆盖原文件；界面显示可理解的错误。

## 界面设计

每个账号行在现有额度信息旁增加紧凑信息：

- `可用重置 2`
- `已用重置 3（本机）`
- `单次周额度 US$40` 或 `单次月额度 US$200`

没有服务器值或用户记录时显示 `—`，不显示错误的 `0`。

账号行提供一个轻量“编辑”入口，用于修改：

- 单次额度美元值。
- 累计已使用重置次数。

金额只允许非负、最多两位小数；次数只允许非负整数。周期名称沿用服务器识别出的 Weekly/Monthly；无法识别时显示“单周期额度”。

如果后端提供 `individualLimit`，界面单独标记为“官方月度上限”，不覆盖用户记录的周/月套餐金额。

## 添加账号流程

点击“+”后：

1. 不再显示 `ConfirmAddAsync` 前置确认 MessageBox。
2. 直接打开现有添加账号 `OperationWindow`。
3. 窗口右上角 `X` 和底部“取消登录”始终可见。
4. 继续使用现有安全登录、取消、认证恢复和 Codex 重启链路。
5. 成功、失败或取消结果在同一个进度窗口中显示，不再增加结果弹窗。

## 刷新与错误处理

- 只有用户点击“刷新额度”时才调用非公开接口。
- 单个账号刷新失败不影响其他账号。
- 重置次数字段缺失时保留为“—”，不报整个额度刷新失败。
- 本机元数据读取失败不影响账号切换和服务器额度刷新。
- 任何接口响应和日志都不得输出令牌或完整认证内容。

## 验证

- 解析器能读取数值和字符串形式的 `available_count`，缺失或无效时返回未知。
- 解析器不会把 `credits.balance` 当作套餐额度。
- 本机元数据可保存、重载并按 `AccountKey` 隔离。
- 无效金额、负数和非整数次数无法保存。
- 点击“+”不再调用前置确认弹窗。
- 添加账号窗口运行中、失败、成功和取消状态都存在可见 `X`。
- 现有安全登录、切换、移除、托盘和额度测试保持通过。

## 验收标准

1. 刷新后可看到各账号当前可用重置次数；后端未返回时显示“—”。
2. 可为每个账号记录并显示累计已用重置次数，标注“本机”。
3. 可为每个账号记录单次周/月额度美元值，赠送重置次数不会改变该值。
4. 官方月度上限与用户记录的套餐额度分开显示。
5. 点击“+”只出现一个带 `X` 的登录进度窗口。
6. 认证文件、对话历史与个性化配置不因本功能发生变化。
